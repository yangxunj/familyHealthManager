## 1. 产品概述

### 1.1 产品定位

家庭健康管理平台是一个面向家庭用户的健康数据管理工具，帮助用户：
- 集中管理所有家庭成员的健康档案
- 数字化存储体检报告、医疗记录等健康文档
- 追踪关键健康指标的变化趋势
- 借助AI获得个性化健康建议和咨询服务

### 1.2 目标用户

| 用户类型 | 典型场景 |
|---------|---------|
| 家庭管理者 | 管理全家人的健康档案，关注父母和孩子的健康状况 |
| 健康关注者 | 系统记录自己的健康数据，追踪身体变化 |
| 慢病患者 | 长期追踪特定指标，获得健康管理建议 |
| 老年人子女 | 远程关注父母的健康状况 |

### 1.3 核心价值

| 价值点 | 说明 |
|-------|------|
| **集中管理** | 一个平台管理全家人的健康数据，告别散乱的纸质报告 |
| **长期追踪** | 记录历史数据，可视化展示健康指标变化趋势 |
| **AI助手** | 智能分析健康数据，提供个性化建议和健康咨询 |
| **便捷存储** | 随时随地查看电子化的体检报告和医疗记录 |
| **风险预警** | 基于数据分析，及时发现潜在健康风险 |

### 1.4 产品目标

**短期目标（MVP）**：
1. 实现家庭成员的基础信息管理
2. 支持健康文档（体检报告等）的上传和管理
3. 提供AI健康建议功能
4. 实现AI健康咨询对话功能

**中期目标**：
1. 健康指标追踪与趋势分析
2. 健康提醒功能（用药、复查等）
3. 家庭成员健康概览看板

**长期目标**：
1. 与医疗机构对接，自动同步体检数据
2. 智能健康风险评估
3. 个性化健康计划生成

## 2. 用户角色与权限

### 2.1 角色定义

本产品采用**单一用户角色**模式，即所有注册用户具有相同权限，但只能管理自己创建的家庭数据。

#### 用户（USER）

**核心职责**：管理自己家庭的所有健康数据

**功能权限**：
- [x] 创建/编辑/删除家庭成员
- [x] 上传/查看/删除家庭成员的健康文档
- [x] 添加/编辑/删除健康记录和指标
- [x] 获取AI健康建议
- [x] 与AI进行健康咨询对话
- [x] 查看健康数据趋势图表
- [x] 设置健康提醒

### 2.2 数据隔离规则

| 数据类型 | 访问规则 |
|---------|---------|
| 家庭成员 | 仅创建者可见和操作 |
| 健康文档 | 仅关联成员的创建者可见 |
| 健康记录 | 仅关联成员的创建者可见 |
| AI对话记录 | 仅发起者可见 |

## 3. 核心功能模块

### 3.1 家庭成员管理

#### 3.1.1 新增家庭成员

**功能描述**：用户可以添加家庭成员，建立健康档案。

**输入字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 姓名 | 文本 | 是 | 家庭成员姓名 |
| 与用户关系 | 选择 | 是 | 本人/配偶/父亲/母亲/儿子/女儿/其他 |
| 性别 | 选择 | 是 | 男/女 |
| 出生日期 | 日期 | 是 | 用于计算年龄 |
| 头像 | 图片 | 否 | 可上传或选择默认头像 |
| 血型 | 选择 | 否 | A/B/AB/O/未知 |
| 身高(cm) | 数字 | 否 | 基础身体数据 |
| 体重(kg) | 数字 | 否 | 基础身体数据 |
| 慢性病史 | 多选 | 否 | 高血压/糖尿病/心脏病/其他 |
| 过敏史 | 文本 | 否 | 药物过敏、食物过敏等 |
| 备注 | 文本 | 否 | 其他需要记录的信息 |

**业务规则**：
- 每个用户可添加的家庭成员数量上限：20人
- 必须先有"本人"角色才能添加其他成员
- 姓名+出生日期不能重复（同一用户下）

#### 3.1.2 编辑家庭成员

**功能描述**：修改家庭成员的基本信息。

**可编辑字段**：所有新增时的字段均可编辑

**业务规则**：
- 编辑后保留历史数据（身高体重变更记录）
- "本人"角色不可删除，但可编辑

#### 3.1.3 删除家庭成员

**功能描述**：删除家庭成员及其所有关联数据。

**业务规则**：
- 删除前需二次确认
- 删除为软删除，数据保留30天后物理删除
- "本人"角色不可删除
- 关联数据（文档、记录、对话）同步标记删除

#### 3.1.4 家庭成员列表

**功能描述**：查看所有家庭成员的概览信息。

**列表信息**：
- 头像
- 姓名
- 关系
- 年龄
- 最近一次体检日期
- 健康状态标签（正常/关注/预警）
- 待办事项数量（如有）

**排序方式**：
- 默认：按关系亲近度排序（本人→配偶→父母→子女→其他）
- 可选：按姓名、年龄排序

### 3.2 健康文档管理

#### 3.2.1 上传健康文档

**功能描述**：为家庭成员上传体检报告、医疗记录等健康相关文档。

**支持的文档类型**：

| 类型 | 文件格式 | 说明 |
|------|---------|------|
| 体检报告 | PDF/图片 | 年度体检、专项体检等 |
| 检验报告 | PDF/图片 | 血常规、尿常规、生化检查等 |
| 影像报告 | PDF/图片 | CT、MRI、X光、B超等 |
| 病历记录 | PDF/图片 | 门诊病历、住院记录等 |
| 处方单 | PDF/图片 | 药物处方 |
| 其他 | PDF/图片/文档 | 其他健康相关文档 |

**输入字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 所属成员 | 选择 | 是 | 选择家庭成员 |
| 文档类型 | 选择 | 是 | 体检报告/检验报告/影像报告/病历记录/处方单/其他 |
| 文档名称 | 文本 | 是 | 如"2026年度体检报告" |
| 检查日期 | 日期 | 是 | 文档对应的检查/就诊日期 |
| 检查机构 | 文本 | 否 | 医院或体检机构名称 |
| 文件 | 文件 | 是 | 支持单个或多个文件，单文件最大20MB |
| 备注 | 文本 | 否 | 补充说明 |

**业务规则**：
- 单次最多上传10个文件
- 支持的图片格式：JPG、PNG、HEIC
- 支持的文档格式：PDF、DOC、DOCX
- 上传后支持AI解析提取关键指标（可选功能）

#### 3.2.2 文档列表与查看

**功能描述**：查看家庭成员的所有健康文档。

**列表信息**：
- 文档名称
- 文档类型标签
- 检查日期
- 检查机构
- 上传时间
- 文件数量

**筛选条件**：
- 按家庭成员筛选
- 按文档类型筛选
- 按时间范围筛选

**查看功能**：
- 图片类型：支持预览、缩放、旋转
- PDF类型：内嵌预览器
- 支持下载原文件

#### 3.2.3 文档删除

**业务规则**：
- 软删除，30天后物理删除
- 删除后关联的AI解析数据同步删除

### 3.3 健康记录管理

#### 3.3.1 添加健康记录

**功能描述**：记录日常健康指标数据，用于长期追踪。

**支持的指标类型**：

| 类别 | 指标 | 单位 | 正常范围（成人参考） |
|------|------|------|---------------------|
| 基础指标 | 身高 | cm | - |
| 基础指标 | 体重 | kg | BMI 18.5-24.9 |
| 基础指标 | 腰围 | cm | 男<90, 女<85 |
| 心血管 | 收缩压 | mmHg | 90-140 |
| 心血管 | 舒张压 | mmHg | 60-90 |
| 心血管 | 心率 | 次/分 | 60-100 |
| 血糖 | 空腹血糖 | mmol/L | 3.9-6.1 |
| 血糖 | 餐后血糖 | mmol/L | <7.8 |
| 血糖 | 糖化血红蛋白 | % | <6.5 |
| 血脂 | 总胆固醇 | mmol/L | <5.2 |
| 血脂 | 甘油三酯 | mmol/L | <1.7 |
| 血脂 | 高密度脂蛋白 | mmol/L | >1.0 |
| 血脂 | 低密度脂蛋白 | mmol/L | <3.4 |
| 其他 | 体温 | ℃ | 36.0-37.3 |
| 其他 | 血氧饱和度 | % | 95-100 |

**输入字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 所属成员 | 选择 | 是 | 选择家庭成员 |
| 记录日期 | 日期时间 | 是 | 默认当前时间 |
| 指标类型 | 多选 | 是 | 可同时记录多个指标 |
| 指标数值 | 数字 | 是 | 根据选择的指标输入对应数值 |
| 测量场景 | 选择 | 否 | 晨起/餐前/餐后/运动后/睡前 |
| 备注 | 文本 | 否 | 特殊情况说明 |

**业务规则**：
- 同一天同一指标可记录多次（如三餐血糖）
- 异常值自动标记（超出正常范围）
- 支持从文档解析自动导入指标数据

#### 3.3.2 健康趋势图表

**功能描述**：可视化展示健康指标的历史变化趋势。

**图表类型**：
- 折线图：展示单一指标随时间的变化
- 组合图：展示相关指标的对比（如收缩压/舒张压）
- BMI计算：根据身高体重自动计算并展示

**筛选条件**：
- 按家庭成员筛选
- 按指标类型筛选
- 按时间范围筛选（近7天/近1月/近3月/近1年/自定义）

**图表功能**：
- 显示正常范围参考线
- 标记异常值点
- 支持数据点详情查看
- 支持图表导出

### 3.4 AI健康建议

#### 3.4.1 功能描述

基于家庭成员的健康档案（基本信息、健康记录、体检报告等），AI分析并生成个性化的健康建议。

#### 3.4.2 建议内容结构

```
┌─────────────────────────────────────────────────────────┐
│                    健康建议报告                          │
│                   [家庭成员姓名]                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  📊 健康概况                                             │
│  ├─ 整体健康评分：85/100                                 │
│  ├─ 主要健康优势：心肺功能良好、体重正常                   │
│  └─ 需关注方面：血压偏高、血脂边缘                        │
│                                                         │
│  ⚠️ 重点关注事项（按优先级排序）                          │
│  1. 血压管理 - 收缩压持续偏高，建议...                    │
│  2. 血脂控制 - 低密度脂蛋白接近临界值...                  │
│  3. ...                                                 │
│                                                         │
│  💡 健康改善建议                                         │
│  ├─ 饮食建议：减少钠盐摄入，增加...                       │
│  ├─ 运动建议：每周进行150分钟中等强度...                  │
│  ├─ 作息建议：保证7-8小时睡眠...                         │
│  └─ 复查建议：建议3个月后复查血压、血脂...                │
│                                                         │
│  📋 建议行动清单                                         │
│  □ 每日测量血压并记录                                    │
│  □ 本周预约心内科门诊                                    │
│  □ 开始低盐饮食计划                                      │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

#### 3.4.3 生成触发方式

| 触发方式 | 说明 |
|---------|------|
| 手动生成 | 用户点击"生成健康建议"按钮 |
| 定期生成 | 每月自动生成一次（可关闭） |
| 事件触发 | 上传新体检报告后自动生成 |

#### 3.4.4 业务规则

- 首次生成需要至少有基本信息和1份体检报告或5条以上健康记录
- 建议内容基于当前数据动态生成，不保存历史版本
- 提供明确的免责声明：AI建议仅供参考，不能替代专业医疗诊断
- 紧急情况（如极端异常值）需提示用户立即就医

### 3.5 AI健康咨询

#### 3.5.1 功能描述

用户可以选择某个家庭成员，与AI进行关于该成员健康问题的对话咨询。

#### 3.5.2 对话场景示例

**场景1：指标解读**
```
用户：我爸爸最近体检，低密度脂蛋白是3.8，这个严重吗？
AI：根据您父亲的体检数据，低密度脂蛋白（LDL-C）为3.8 mmol/L，
    超过了正常参考值上限（<3.4 mmol/L）。结合他的其他情况：
    - 年龄：65岁
    - 有高血压病史
    - 总胆固醇也偏高
    建议：
    1. 这个水平属于边缘升高，需要引起重视
    2. 建议3个月后复查血脂四项
    3. 饮食上建议减少饱和脂肪摄入...
    4. 如持续升高，可能需要药物干预，建议咨询心内科医生
```

**场景2：症状咨询**
```
用户：我妈妈最近总说头晕，会不会跟血压有关？
AI：根据您母亲的健康档案，她确实有高血压病史。
    头晕可能与血压有关，但也有其他可能原因。

    查看她近期的血压记录：
    - 1月15日：145/95 mmHg（偏高）
    - 1月18日：138/88 mmHg（边缘）
    - 1月20日：152/98 mmHg（偏高）

    血压波动较大，可能是头晕的原因之一。建议：
    1. 增加血压监测频率，每天早晚各测一次
    2. 检查是否按时服用降压药
    3. 如头晕持续或加重，建议尽快就医排除其他原因...
```

**场景3：用药咨询**
```
用户：我最近在吃降压药，能不能吃西柚？
AI：这是个很好的问题！很多降压药确实与西柚有相互作用。

    根据您的用药记录（如有），我需要确认您使用的具体药物。
    常见与西柚有相互作用的降压药包括：
    - 钙通道阻滞剂（如氨氯地平、硝苯地平）
    - 部分他汀类药物

    建议您告诉我具体药名，或咨询您的主治医生/药师。
```

#### 3.5.3 对话上下文

AI对话时可访问的上下文信息（需用户授权）：
- 当前选中成员的基本信息（年龄、性别、病史等）
- 最近的健康记录数据
- 最近上传的体检报告关键指标
- 历史对话记录（本次会话）

#### 3.5.4 业务规则

- 每次对话需选择关联的家庭成员
- 对话记录保存，支持查看历史
- 提供常用问题快捷入口
- 明确免责声明：AI仅提供健康科普，不进行诊断
- 识别紧急情况关键词，提示用户及时就医

- avatar, bloodType
- height, weight

- value, unit

- notes
- createdAt

- id, memberId（所属成员）

- generatedAt

#### ChatSession（咨询会话）
- id, userId, memberId
- title
- createdAt, updatedAt

#### ChatMessage（对话消息）
- id, sessionId
- role（user/assistant）
- content
- createdAt

## 5. 页面流程设计

### 5.2 核心页面流程

#### 5.2.1 新用户引导流程

```
注册/登录 → 创建"本人"档案 → 添加家庭成员（可选）→ 上传首份体检报告（可选）→ 进入首页
```

#### 5.2.2 上传体检报告流程

```
选择成员 → 选择文档类型 → 上传文件 → 填写信息 → 确认上传 → [可选]AI解析提取指标
```

#### 5.2.3 获取AI建议流程

```
进入成员详情 → 点击"获取AI建议" → AI分析数据 → 展示建议报告 → 可导出/分享
```

#### 5.2.4 AI咨询对话流程

```
进入AI咨询 → 选择关联成员 → 开始对话/继续历史会话 → 输入问题 → AI回复 → 继续对话
```

## 6. 业务规则

### 6.1 数据完整性规则

| 规则 | 说明 |
|------|------|
| 必有本人 | 用户必须先创建"本人"角色的家庭成员 |
| 成员上限 | 每个用户最多创建20个家庭成员 |
| 文件大小 | 单个文件最大20MB，单次上传最多10个文件 |
| 软删除 | 删除操作为软删除，保留30天 |

### 6.2 健康指标规则

| 规则 | 说明 |
|------|------|
| 参考范围 | 根据年龄、性别动态调整正常参考范围 |
| 异常标记 | 超出正常范围的值自动标记为异常 |
| 极端值预警 | 极端异常值（如血压>180或<70）触发紧急提示 |
| 数据验证 | 输入值需在合理范围内（如体重0-500kg） |

### 6.3 AI功能规则

| 规则 | 说明 |
|------|------|
| 数据要求 | 生成AI建议需至少有基本信息+1份报告或5条记录 |
| 免责声明 | AI功能页面必须展示免责声明 |
| 紧急识别 | 识别紧急情况关键词，提示就医 |
| 上下文隔离 | AI对话仅能访问当前选中成员的数据 |

### 6.4 隐私与安全规则

| 规则 | 说明 |
|------|------|
| 数据隔离 | 用户只能访问自己创建的数据 |
| 文件加密 | 上传的健康文档加密存储 |
| 会话超时 | 登录会话30天有效，敏感操作需重新验证 |
| 操作日志 | 关键操作记录审计日志 |

## 7. AI功能设计

### 7.1 AI能力边界

**AI可以做的**：
- 解读体检报告指标的含义
- 提供健康生活方式建议
- 分析健康数据趋势
- 回答常见健康问题
- 提供就医科室建议

**AI不能做的**：
- 做出医学诊断
- 开具处方或建议用药剂量
- 替代专业医生的判断
- 处理紧急医疗情况

### 7.2 AI Prompt策略

#### 健康建议生成Prompt结构

```
你是一位专业的健康管理顾问，请根据以下用户健康档案生成个性化健康建议。

## 用户健康档案
- 基本信息：{姓名}，{性别}，{年龄}岁
- 身高体重：{身高}cm，{体重}kg，BMI={计算值}
- 既往病史：{慢性病列表}
- 过敏史：{过敏信息}

## 最近健康数据
{最近30天的健康记录摘要}

## 最新体检报告关键指标
{体检报告解析后的关键指标}

## 生成要求
1. 分析整体健康状况，给出健康评分（0-100）
2. 列出需要重点关注的健康问题（按优先级排序）
3. 针对每个问题给出具体可行的改善建议
4. 提供饮食、运动、作息方面的综合建议
5. 列出建议的复查项目和时间

## 重要提醒
- 建议应具体、可执行
- 不要做出医学诊断
- 对于异常指标，建议就医而非自行处理
- 语气温和专业，易于理解
```

#### 健康咨询对话Prompt结构

```
你是一位专业的健康顾问，正在为用户解答关于家庭成员的健康问题。

## 当前咨询对象
- 姓名：{成员姓名}
- 与用户关系：{关系}
- 基本信息：{性别}，{年龄}岁
- 既往病史：{慢性病列表}

## 可参考的健康数据
{相关健康记录和体检数据}

## 对话规则
1. 基于用户提供的信息和系统中的健康数据回答
2. 回答应准确、专业、易懂
3. 涉及具体治疗建议时，引导用户咨询医生
4. 识别紧急情况，提示用户及时就医
5. 可以主动询问更多信息以便更好地回答

## 回复格式
- 使用简洁清晰的语言
- 必要时使用列表格式
- 给出具体可行的建议
- 在回复末尾可添加相关追问引导
```

### 7.3 AI调用流程

```
用户请求 → 权限验证 → 构建上下文 → 调用LLM API → 后处理 → 返回结果
                          ↓
                    - 获取成员信息
                    - 获取相关健康数据
                    - 构建Prompt
                    - 添加安全护栏
```
